# 问卷调研方法与关键指标解读

## Survey Research Methods & Metrics Guide

本文件是 `survey-research-skill.md` 的补充说明，用于解释在问卷调研分析表格中常用的调研方法、统计方法，以及每类方法中最值得关注的数据指标。

---

## 一、结构化问卷调研法

结构化问卷调研是整个分析的底层方法。

它的作用是通过统一问题收集用户的自报信息，再基于统计方法观察用户结构、使用场景、产品偏好、购买动机、渠道行为和潜在人群分层。

### 适合回答的问题

- 用户是谁？
- 用户如何使用产品？
- 用户处于什么经验和技能阶段？
- 用户为什么选择或继续使用产品？
- 用户后续有什么购买或升级可能？
- 用户在哪里被内容影响？
- 用户在哪里购物？

### 最值得关注的数据

| 指标 | 说明 |
|---|---|
| N | 进入分析的样本量 |
| 样本口径 | 样本来自哪里，是否为问卷自报用户 |
| 去重规则 | 是否按邮箱、联系方式或唯一 ID 去重 |
| 来源结构 | 数据是否来自多个问卷、渠道或社群 |
| 样本限制 | 是否能代表全量用户，是否存在招募偏差或平台偏差 |

### 注意事项

不得把问卷受访样本直接表述为全量市场用户或全量购买用户。

推荐表述：

> 本报告基于问卷中自报拥有或使用某产品的受访者样本，不等同于全量购买用户。

---

## 二、数据清洗与编码方法

数据清洗与编码是正式统计分析前的基础步骤。

### 使用内容

- 单选题标准化
- 等级变量数值编码
- 多选题 one-hot 编码
- Others 文本归并
- 开放题主题编码
- 缺失值检查
- 异常值检查
- 逻辑冲突检查

### 最值得关注的数据

| 指标 | 说明 |
|---|---|
| Missing rate | 字段缺失率，决定该字段能否用于核心结论 |
| Valid N | 每个题目真实参与统计的人数 |
| One-hot accuracy | 多选题是否正确拆成 0/1 字段 |
| Others normalization | 手写答案是否被合理归并 |
| Logic flags | 是否存在前后回答矛盾 |

### 注意事项

清洗不是为了删除“不好看”的数据，而是为了保证每个统计结果可复核。

多选题必须拆成 one-hot 字段，不能当作单选题处理。

---

## 三、描述性统计分析

描述性统计是第一层正式分析。

它回答的是：

> 这批受访者的样本结构是什么？

### 常见输出

- 年龄分布
- 国家 / 地区分布
- 职业分布
- 家庭结构
- 收入区间
- 教育程度
- 使用年限
- 使用频率
- 技能等级
- 当前设备满足程度
- 购买 / 升级计划
- 预算区间
- 使用场景选择率
- 功能偏好选择率
- 平台和购物渠道分布

### 最值得关注的数据

| 指标 | 说明 |
|---|---|
| Count | 选择该选项的人数 |
| Percentage | 选择该选项的人数占有效样本比例 |
| Valid N | 当前题目的有效样本量 |
| Missing N | 当前题目的缺失样本量 |
| Top ranking | 高频选项排序 |

### 注意事项

描述性统计只描述事实，不解释原因。

多选题的百分比总和可以超过 100%。

---

## 四、使用场景规则打分法

使用场景不能只靠单题判断，应使用多个题目互相验证。

例如，判断“小生意 / 副业场景”时，可以综合：

- 使用场景：Side hustle / Small business
- 使用目的：Business & Entrepreneurship
- 购物渠道：Etsy
- Lifestyle Tag：Small Business Dreamer

### 场景信号强度

| 信号强度 | 判断规则 |
|---|---|
| weak signal | 只在一个相关字段命中 |
| medium signal | 在两个相关字段命中 |
| strong signal | 在三个或以上相关字段命中 |

### 最值得关注的数据

| 指标 | 说明 |
|---|---|
| Scenario score | 场景命中分数 |
| Strong scenario count | 强场景用户数量 |
| Weak / medium / strong ratio | 场景稳定性 |
| Supporting variables | 支撑该场景的题目来源 |

### 注意事项

场景打分属于规则型用户标签建模，不是传统统计检验。

它的价值是减少主观命名画像的风险。

## 五、产品功能优先级分析

产品功能优先级分析用于回答：

> 用户选择产品时到底更看重什么？

### 常见维度

- 价格 / 性价比
- 易用性
- 安全性
- 售后保障
- 生产效率
- 精度 / 深度
- 材料支持
- 维护成本
- 软件体验
- 社区支持
- 便携性
- 外观设计
- 品牌影响力

### 最值得关注的数据

| 指标 | 说明 |
|---|---|
| Count | 选择某因素的人数 |
| Percentage | 某因素在样本中的覆盖率 |
| Top 10 ranking | 产品卖点优先级排序 |
| Segment difference | 不同用户层级或场景之间的差异 |

### 注意事项

如果原题限制 Top 3，但数据存在超选，解释时应避免说“绝对前三优先级”，更适合表述为“高频选择因素”。

---

## 六、交叉分析法

交叉分析用于比较两个变量之间的分布关系。

它回答的是：

> 不同类型用户之间，选择分布是否不同？

### 常见分析组合

- 技能等级 × 使用频率
- 使用场景 × 功能偏好
- 预算区间 × 购买计划
- 家庭结构 × 使用场景
- 常用平台 × 购物渠道

### 最值得关注的数据

| 指标 | 说明 |
|---|---|
| Count | 每个交叉格子中的人数 |
| Row % | 某类用户内部的比例 |
| Column % | 某个选项下各类用户的构成 |
| Cell size | 单个格子的样本量是否足够 |

### 注意事项

交叉表只能看分布差异，不能单独证明差异是否显著。

如需判断统计关联，需要结合卡方检验。

---

## 七、卡方检验 Chi-square Test

卡方检验用于判断两个分类变量之间是否存在统计关联。

### 适合的问题

- 某类场景用户是否更关注某个功能？
- 不同技能等级用户是否有不同功能偏好？
- 不同预算用户的升级计划是否不同？
- 不同平台用户的购物渠道是否不同？

### 最值得关注的数据

| 指标 | 说明 |
|---|---|
| p-value | 判断两个变量是否存在统计显著关联 |
| Cramer’s V | 判断分类变量之间的关联强度 |
| N | 样本量是否足够 |
| Minimum expected count | 是否满足卡方检验条件 |

### p-value 解读

| p-value | 解读 |
|---|---|
| p < 0.05 | 可认为两个变量之间存在统计关联 |
| p < 0.01 | 关联更显著 |
| p ≥ 0.05 | 没有足够证据说明有关联 |

### Cramer’s V 解读

| Cramer’s V | 解读 |
|---|---|
| < 0.1 | 很弱 |
| 0.1–0.3 | 弱关联 |
| 0.3–0.5 | 中等关联 |
| ≥ 0.5 | 强关联 |

### 注意事项

p-value 只说明是否显著，不说明关联强度。

更值得关注的是：

> p-value 显著 + Cramer’s V 不低 + 样本量足够 + 期望频数满足条件。

即使 p-value 显著，如果 Cramer’s V 很低，也只能说明关联存在但强度有限。

---

## 八、Spearman 等级相关分析

Spearman 相关分析适用于有顺序的变量。

问卷数据中很多变量不是连续数值，而是等级变量，所以通常不适合直接用 Pearson r，更适合使用 Spearman ρ。

### 适合的变量

- 使用年限
- 使用频率
- 技能等级
- 当前设备满足程度
- 预算区间
- 收入区间
- 购买 / 升级计划

### 最值得关注的数据

| 指标 | 说明 |
|---|---|
| Spearman ρ | 等级变量之间的相关方向和强度 |
| p-value | 相关是否显著 |
| N | 参与计算的样本量 |
| 正负号 | 正相关或负相关 |

### Spearman ρ 解读

| ρ 绝对值 | 解读 |
|---|---|
| < 0.1 | 几乎无相关 |
| 0.1–0.3 | 弱相关 |
| 0.3–0.5 | 中等相关 |
| ≥ 0.5 | 强相关 |

### 正负号解读

| 符号 | 含义 |
|---|---|
| 正数 | 一个变量越高，另一个变量也倾向越高 |
| 负数 | 一个变量越高，另一个变量倾向越低 |

### 注意事项

Spearman 相关不是因果关系。

例如，技能等级和使用频率相关，不能直接写成“因为技能高所以使用频率高”。也可能是使用频率提升了技能，或两者都受到使用年限影响。

## 九、多选题共现矩阵

共现矩阵用于观察：

> 哪些选项经常被同一批用户一起选择？

### 适合的问题

- 礼品场景和纯 DIY 是否经常共现？
- 小生意和 Etsy 是否经常共现？
- 安全优先级和 Class 1 Safety 是否共现？
- YouTube 和教程资产库是否共现？

### 最值得关注的数据

| 指标 | 说明 |
|---|---|
| Co-occurrence count | 两个选项同时出现的人数 |
| Co-occurrence rate | 同时出现占总样本比例 |
| 高频共现组合 | 可以提示主流用户场景组合 |
| 低频但高相关组合 | 可以作为小众洞察线索 |

### 注意事项

共现矩阵适合发现组合关系，但本身不判断显著性。

如果要判断共现是否高于随机，需要进一步使用关联规则分析或卡方检验。

---

## 十、关联规则分析

关联规则分析用于多选题之间的共现关系挖掘。

它回答的是：

> 选择 A 的用户，有多大概率也会选择 B？这种共现是否高于随机？

### 常见分析问题

- 选择 Pet Parent 的用户，还常选择哪些使用场景？
- 选择 Small Business Dreamer 的用户，还常关注哪些功能？
- 选择 YouTube 的用户，是否更常关注 tutorials / creative asset library？
- 选择 Home Decor 的用户，是否更常选择 Pinterest？
- 选择 Safety 的用户，是否也选择 Class 1 / enclosed / smoke concern？

### 三个核心指标

| 指标 | 含义 |
|---|---|
| Support | A 和 B 同时出现的比例 |
| Confidence | 选择 A 的用户中，有多少也选择 B |
| Lift | A 和 B 的共现是否高于随机 |

### Support 解读

Support 表示覆盖面。

例如 support = 0.15，表示 15% 的样本同时选择了 A 和 B。

Support 太低时，即使 Lift 很高，也只能作为小众信号。

### Confidence 解读

Confidence 是条件概率。

例如 confidence = 0.70，表示选择 A 的用户中，70% 也选择了 B。

它适合判断：

> 如果用户属于 A，那么他是否大概率也有 B 特征？

### Lift 解读

| Lift | 解读 |
|---|---|
| ≈ 1 | 接近随机共现 |
| > 1 | 正向共现 |
| > 1.2 | 有一定关联信号 |
| > 1.5 | 较强共现信号 |
| < 1 | 低于随机共现 |

### 最可靠的规则

更值得关注的是：

> Support 不低 + Confidence 高 + Lift > 1.2 或 1.5。

如果 Lift 很高但 Support 很低，说明它可能只是少数人的强烈信号，不能作为主流画像依据。

## 十一、开放题文本内容分析

开放题用于补充用户原始表达中的痛点和需求。

### 常见流程

1. 保留用户原始文本
2. 建立主题标签
3. 对每条回答进行多标签编码
4. 统计各主题出现频次
5. 标记不确定回答
6. 抽样人工复核
7. 提取典型原话作为辅助证据

### 常见主题

- ease of use
- auto focus
- safety
- smoke control
- speed
- precision
- software
- camera alignment
- maintenance
- price
- material compatibility
- aftersales

### 最值得关注的数据

| 指标 | 说明 |
|---|---|
| Theme count | 某主题出现多少次 |
| Theme percentage | 某主题在开放题中的占比 |
| Example response | 典型原话证据 |
| Coding confidence | 编码是否明确 |
| needs_review | 是否需要人工复核 |

### 注意事项

开放题可以补充“为什么”，但不能替代定量统计。

如果开放题主题高频出现，并且选择题中也有对应高频选项，则该信号可信度更高。

---

## 十二、用户分群 / 聚类分析

用户分群用于探索样本中是否存在相似用户群体。

### 可使用方法

- One-hot 多选题后聚类
- K-modes
- K-prototypes
- 层次聚类
- MCA 多重对应分析

### 常用变量

- 使用场景
- 使用目的
- 使用频率
- 技能等级
- 功能偏好
- 购买优先级
- 预算
- 平台偏好
- 购物渠道
- Lifestyle Tags

### 最值得关注的数据

| 指标 | 说明 |
|---|---|
| Cluster size | 每个群体有多少人 |
| Cluster share | 每个群体占总样本比例 |
| Key over-index features | 该群体显著高于总体的特征 |
| Key under-index features | 该群体显著低于总体的特征 |
| K sensitivity | 分成几个群时结果是否稳定 |
| Small cluster warning | 小群体是否样本过少 |

### 注意事项

聚类结果是探索性分群，不是天然真实存在的人群。

Persona 名称必须来自该群体稳定特征，而不是先起名字再找数据证明。

## 十三、来源偏差与样本质量验证

来源偏差检查用于判断不同数据来源是否能合并分析。

### 常见检查维度

- 年龄分布
- 国家分布
- 使用频率
- 技能等级
- 使用场景
- 预算区间
- 购买计划
- 功能偏好

### 最值得关注的数据

| 指标 | 说明 |
|---|---|
| Source count | 每个来源的样本量 |
| Source distribution | 不同来源样本结构是否接近 |
| Missingness by source | 某来源是否缺失更多 |
| Key variable difference | 不同来源在关键变量上是否差异明显 |

### 注意事项

如果来源差异明显，报告中应说明：

> 合并结果代表问卷样本整体，但不同来源可能存在结构差异。

必要时应分来源展示。

## 十四、最值得重点关注的指标总表

| 指标 | 出现在哪种方法 | 为什么重要 |
|---|---|---|
| N / Valid N | 所有统计 | 决定样本基础是否可靠 |
| Percentage | 描述统计 | 判断某选项覆盖面 |
| Missing rate | 质量检查 | 判断字段能否作为核心结论 |
| p-value | 卡方 / Spearman | 判断关联是否显著 |
| Cramer’s V | 卡方检验 | 判断分类变量关联强度 |
| Spearman ρ | 等级相关 | 判断等级变量相关方向和强度 |
| Support | 关联规则 | 判断组合关系覆盖面 |
| Confidence | 关联规则 | 判断条件概率 |
| Lift | 关联规则 | 判断共现是否高于随机 |
| Theme count | 文本内容分析 | 判断开放题主题高频程度 |
| Cluster share | 聚类分析 | 判断画像群体规模 |

---

## 十五、一句话记忆

> 比例看规模，p-value 看显著，Cramer’s V 看分类关联强度，Spearman ρ 看等级相关强度，Support 看覆盖面，Confidence 看条件概率，Lift 看是否高于随机共现。

---

## 十六、汇报时的推荐表达

可以这样和团队解释：

> 本次问卷分析采用结构化问卷调研作为基础方法，先完成样本口径验证、数据清洗、字段编码和质量检查，再通过描述性统计回答样本结构，通过交叉表和卡方检验判断分类变量之间的关联，通过 Spearman 相关分析处理等级变量关系，通过共现矩阵和关联规则挖掘多选题组合关系，通过开放题文本内容分析补充用户原始痛点表达，最后在基础统计和关联分析稳定后进行探索性用户分群。

> 所有结论均需要同时关注样本量、比例、显著性、关联强度和样本限制，避免把相关性误写成因果关系，避免把问卷受访者结果扩大为全量用户结论。
